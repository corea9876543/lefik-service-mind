# 작업 규칙 (이 저장소의 모든 세션에서 항상 지킬 것)

## 1. 코드·스크립트는 올리기(커밋/푸시) 전에 실제로 실행해서 검증하고 통과시킨다
- 커밋/푸시 전에 해당 코드를 이 환경에서 **직접 실행**하거나 최소한 문법·동작을 검사해 **통과**시킨다.
  - JavaScript/Node: `node --check`, 가능하면 실제 실행 또는 테스트
  - 셸 스크립트: `bash -n` + 가능하면 실제 실행(임시 환경에서 왕복 테스트)
  - JSON/설정 파일: 파서(`python3 -c json.load` 등)로 검증
- **이 리눅스 샌드박스에서 실행이 불가능한 경우**(예: 윈도우 전용 `.bat`, 실제 안경/하드웨어·외부 계정·시크릿 필요)에는
  "실행 검증을 못 했다"는 사실을 **솔직히 밝히고**, 대체 검증(문법 검사·ASCII 검사·수동 추적)을 한 뒤
  남는 리스크를 명시한다. 추측으로 "검증됨"이라고 말하지 않는다.

## 2. 모든 답변을 보내기 전에 오류가 없는지 한 차례 검토하고 통과시킨다
- 답변 전송 전에 **사실관계·명령어·경로·코드·단계 순서**가 맞는지 스스로 한 번 점검한다.
- 불확실하거나 모르는 것은 단정하지 않는다 — 확인하거나, 불확실하다고 명시한다.
- 특히 외부 사실(API/제품 사양 등)은 기억에 의존하지 말고 근거를 확인한다.
